# Harness 评分规则

本规则用于评估一次 harness 运行。优先使用确定性检查，而不是 LLM 主观打分。

| 维度 | 权重 | 确定性信号 |
| --- | ---: | --- |
| 流程完整性 | 22 | 必需阶段产物存在 |
| 产物质量 | 15 | 必需章节、具体文件和命令引用存在 |
| 代码正确性 | 22 | 编译/静态检查和测试通过 |
| 效率 | 10 | 耗时和重复失败循环有记录 |
| 安全合规 | 8 | 没有禁用操作，豁免有记录 |
| 迭代能力 | 5 | 失败检查后有修复和重跑 |
| 验收验证 | 18 | 存在 ATDD、接口或验收证据 |

## 规则

缺失产物给 0 或失败状态。不要给空输出中性分。
