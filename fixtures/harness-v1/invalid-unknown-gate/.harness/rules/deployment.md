# 部署规则

目的：让部署明确、可回滚、可举证。

## 必须执行

1. 部署前识别目标环境。
2. 中高风险部署前确认回滚路径。
3. 记录命令、环境、版本和结果。
4. 部署后执行接口或冒烟测试。
5. 只有部署和接口检查都有记录后，`G7_PRERELEASE` 才能标记为 PASS。

## 避免

- 不要自主部署到生产环境。
- “部署命令已提交”不等于部署成功。
- 高风险任务不得跳过回滚说明。

## 输出格式

```markdown
# 预发部署

- 环境：
- 版本或 commit：
- 命令：
- 结果：
- 回滚：
- 冒烟/接口测试：
- 证据链接或日志：
```
