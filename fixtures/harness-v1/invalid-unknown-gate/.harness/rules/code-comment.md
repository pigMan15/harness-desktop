# 代码注释规则

目的：确保编码阶段生成的代码包含中文注释，便于团队维护和业务理解。

## 必须执行

1. 编码阶段新增或修改的核心逻辑，必须包含必要的中文注释。
2. 涉及业务规则、状态流转、异常处理、数据组装、权限判断、外部系统交互时，必须用中文注释说明原因或约束。
3. 如果方法、类、组件、复杂条件分支是本次新增的，应优先补充中文注释说明用途。
4. 注释应解释“为什么这样做”或“业务约束是什么”，不要只复述代码表面含义。
5. 在 `state.phase_dir/11-development.md` 中记录本次新增/修改代码注释的范围。

## 避免

- 不要写无意义注释，例如“给变量赋值”“调用方法”“返回结果”。
- 不要为了满足规则给每一行都加注释。
- 不要用英文注释替代中文注释，除非项目已有固定英文 API 文档格式；即便如此，业务约束仍应补充中文说明。

## 示例

推荐：

```java
// 下架前先校验全检单状态，避免已完成或作废单据重复触发库存动作。
validateInspectionStatus(order);
```

不推荐：

```java
// 调用校验方法
validateInspectionStatus(order);
```
