# TDD 上下文

当行为发生变化，或风险为中/高时，使用 TDD。

## 循环

1. 编写或定位最小失败测试。
2. 运行它，确认失败原因符合预期。
3. 做最小实现改动。
4. 运行聚焦测试。
5. 运行更广范围的相关测试。
6. 记录命令和结果。

## 测试困难时

如果项目没有测试框架，创建最小有用验证：

- 静态类型检查。
- 脚本化复现。
- CLI 冒烟命令。
- API 请求。
- 人工验证清单。

把限制记录为豁免；不要把它说成通过了单元测试。

## 输出

加入当前阶段产物：

```markdown
## TDD 记录

- 新增或选中的测试：
- 初始失败：
- 实现：
- 聚焦结果：
- 扩展结果：
```
