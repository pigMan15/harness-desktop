# 代码搜索规则

目的：用更少上下文找到正确代码。

## 必须执行

1. 文本和文件发现优先使用 `rg` 与 `rg --files`。
2. 先搜精确标识符，再搜宽泛概念。
3. 定位候选文件后，再读取附近代码。
4. 对结构化数据或代码，优先使用可感知结构的工具。
5. 重要搜索发现要写入当前阶段产物。

## 避免

- 能读局部范围时，不要整文件加载大文件。
- 不要只凭名字推断 API 行为。
- 找到足够可靠的本地模式后，不要继续无边界搜索。

## 常用命令

```powershell
rg --files
rg "ExactIdentifier"
rg "functionName|className" -n
```
