# 分支卫生规则

目的：让流程变更可审计，并避免误覆盖用户工作。

## 必须执行

1. 编辑前检查 `git status --short`。
2. 不要回退不是你改的文件。
3. 需要创建分支时，优先使用带 `codex/` 前缀的描述性分支名。
4. 部署或集成前，确认分支基于预期主干。
5. 将分支和 worktree 操作记录到 `state.phase_dir/09-branch.md` 与 `state.phase_dir/10-worktree.md`。

## 避免

- 除非用户明确要求，不使用 `git reset --hard`。
- 除非用户明确要求并确认，不强推。
- 不清理无关文件。

## 输出格式

```markdown
# 分支记录

- 起始分支：
- 工作区状态：
- 创建分支：
- 主干同步状态：
- 备注：
```
