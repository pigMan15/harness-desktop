# 上下文预算规则

目的：保持主会话聚焦，避免规则过载。

## 加载顺序

1. 入口文件：`AGENTS.md` 或 `CLAUDE.md`
2. 状态：`.harness/state.json`
3. 流程：`.harness/workflow.yaml`
4. 当前角色文件
5. 当前规则文件
6. 当前上下文模板
7. 当前任务所需源码

## 必须执行

- 只读取当前阶段必要文件：当前命令、`state.json`、`workflow.yaml`、dispatcher、当前角色、必要规则/上下文和相关阶段产物。
- 不要预读整套 `.harness/`。
- 不要扫描无关历史 run；恢复旧 run 时只读取目标 `run_id` 对应的 `state.phase_dir` 和 `.harness/runs/<run_id>/state.json`。
- 只有阶段需要时才加载深层上下文。
- 将大型工具输出总结进阶段产物。
- 除非决策需要，不把已完成阶段细节长期留在对话中。
- 用文件当记忆，不用聊天滚动条当记忆。

## 风险信号

- 每轮都重新读取全部规则。
- 工具输出淹没对话。
- 当前阶段不清楚。
- 完成判断依赖记忆而不是产物。
